1.1研究的背景

84

现在的社会经济和网络发展越来越快，喜欢在手机购物的人也愈来愈多，传统的到门店购物的方式已经满足不了人们追求更加便捷的购物心理， 现在的人们更倾向于能够在足不出户的网上购、节时、省心和高效的物流体验。虽然各种实体店超市给我们生活带来了很多便利，不可避免的是这些实体店超市也有各种各样的缺点，如：每天到要处理入库的大量商品导致其效率低下，时刻都需要有人结算付款，人工的速度有限并且会有细微的差错，每次有活动的商品都要重新打印价签，其中的效率一定不会太高。另外一方面，对于一个有稍微有规模的超市来说，每天都会有商品的入库、销售和更新等信息要处理，而这些需要一些有经验的管理员来进行决策、分析和处理，则需要大量人力物力，造成更多的成本。 所以这一切操作得需要相对应的app软件和后台商品的管理系统来操作这一切频繁而又繁琐的事务。

85

在移动互联网时代，人口对企业来说突显格外得重要。人口是指用户接入互联网的门户，成为人口就可以获得大量得用户。经过多年的互联网发展、竞争和淘汰，现在逐渐成为了两个主要的人口：Native App 、Webapp、 Native App 在移动应用向企业推广的过程中首先成为一种趋势。所以企业在探索发展移动信息花时，常用Native App原生的app开发模式。 但是随着使用该 app的时间长久，企业就必须要投入大量的人力和资金去维护和更新迭代，这将逐渐变成一种拖累甚至是一种不必要的负担，所以 Native App原生应用开发模式很难再继续走下去。

86

1.2研究目的和意义

87

在移动互联网方面企业很多从尝试和摸索开始走合适自己的道路，尽可能降低减少开发的成本和研发的周期，如果不能尽早更新迭代则很难持续地保留用户量，所以需要以迭代的方式在短周期内不断完善 Native App的功能， 以满足市场和用户日新月异的需求。但是 Native App 对企业来说，无论以什么样的方式，外包或者自己开发，都会遇到开发周期长、更新滞后、维护成本高昂的问题。所以真正想发展移动信息化的企业都认识到要避免Native App 的负面影响，就必须放弃Native App 原生应用的开发模式，寻找别的更合适的方法。基于HTML5 的网页能够允许Native App 可以实现的功能直接通过网页实现，也就是实现Web App 的开发模式。 HTML5技术可以降低因为不同系统兼容问题带来的各种成本，只需要基于 HTML5的标准开发一套 Web App就可以适配不同平台、不同终端，有了这样的便利之后，企业和开发者可以大幅度缩短对用户及市场的新特性及需求的相应时间；对于用户来说，基 于 HTML5 的 Web App 可以使用 户免于频繁的下载不同的Native App 以及同一个 Native App 不同的版本。所以，虽然到目前为止甚至短期内 Web App还不能完全代替 Native App原生应用的开发模式，但是推动互联网的发展是其开放、自由的属性， 基于 HTML5的 Web APP将会成为主流基于 HTML5的。

88

1.3研究的内容

89

作为一个Web app电商平台，它要求具有简单性、易操作性、准确性、安全性。切实从根本上改变传统的物品交易方式，为大众提供一个便捷简单的操作平台， 大大提高物品交易的高效性，同时提供一个完备个后台管理，为整个交易过程保驾护航，提高平台的信息管理，维护信息的真实性、完整性、实时性。 在构建开发环境时，采用的是微软的 Visual Studio Code开发，开发语言使用 HTML5+ CSS3+ JavaScript+ sass+ Vue+ node. js进行开发， 数据库使用 MongoDB。将采用 B/ S结构， B/ S最大的优点就是可以在任何地方进行操作而不用安装任何专门的软件，只要有一台能上网的手机就能使用，客户端零安装、零维护。

90

Web APP将会成为主流网上交易平台是一个 B/ S系统，根据具体的功能，前台可以分为用户注册模块、用户登录模块、物品查询与浏览模块、物品购买模块， 用户可在页面上进行物品的交易和操作。后台可分为管理员管理模块、用户管理模块、商品信息管理模块、分类信息管理模块、系统管理模块，管理员在后台可以对数据进行增、删、查、改，还可以对数据库进行操作。

2.1HTML5技术

101

经过了8年的不断修订与完善，2014年10月28号， W3 C发布 HTML5的正式推荐标准，实际上，在此之前， HTML5已经在大量网站应用中得到了使用。提供了更符合开发者习惯的新元素及新标签，大大方便了网站的开发，同时使得越来越多的应用被开发出，这说明 HTML5使用及相关技术，已经能够在一些领域替代传统的应用。

102

HTML5能够得到快速发展，是因为他带来的新特性能够实现很多复杂的功能，并大大降低开发难度。他提供本地存储，之前的网页与应用中，对于存储类问题一般是使用Cookie来解决。但是Cookie一般有数量和容量的限制（一般为4K）这大大增加了程序设计的复杂度。HTML5在Windows对象中内置了localStorage和sessionStorage这两个属性，作为本地存储的载体。localStoragelocalStorage生命周期是永久，这意味着除非用户显示在浏览器提供的UI上清除localStorage信息，否则这些信息将永远存在。存放数据大小为一般为5MB，而且它仅在客户端（即浏览器）中保存，不参与和服务器的通信。sessionStorage仅在当前会话下有效，关闭页面或浏览器后被清除。存放数据大小为一般为5MB，而且它仅在客户端（即浏览器）中保存，不参与和服务器的通信。
